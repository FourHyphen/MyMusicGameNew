 - ストーリーメモ
   - 残す目的：良いストーリー/悪いストーリーの勉強用
   - 「誰が」「何をすると」「結果どうなる」「何のためにこの機能が存在する」まで書くべき
     - 自分用なのでそこまでしない
   - ストーリーを実装する際はタスクに分割する
     - 本来はイテレーションプランニングにて行う
 - 疑問点
   - UIに関する記載("テキスト表示とする"とかの要望・顧客との話し合い結果)はストーリーに含めるものなのか？
     - 多分、そのストーリーの達成条件という形で併記すれば良さそうな気がする
 - 画面立ち上げ後
   - [x] 画面立ち上げ後、設定ファイルから読み込んだ曲リストが表示される
     - 少なくともjson形式をサポートすること
   - [x] 画面立ち上げ後、曲を選択するようプレイヤーに何かしらの手段で伝える
     - 手段：画面表示とする
   - [x] 曲リストから曲を選択しない状態でゲームスタートボタンを押すと、ゲームが始まらない
     - この際、何も表示(プレイヤーに通知)しないでOK
   - [x] 曲リストから曲を選択する際、これまでのベスト結果を表示する
   - [ ] キー入力の割り当てを自由に変更できること
 - ゲーム開始時
   - [x] 曲リストから曲を選択した状態でゲームスタートボタンを押すと、ゲームが始まる
     - その際、何かしらの手段でプレイヤーにゲームが始まった旨を伝える
     - 手段：画面表示とする
   - [x] ゲーム開始時、プレイヤーが選択した曲を流し始める
   - [ ] プレイヤーは、判定ラインの形状とノートが降ってくる方向を定めたモードをゲーム開始時に設定できる
     - これはストーリーとして大きすぎる？少なくともモード毎にストーリーを定めた方がやりやすい
   - [ ] プレイヤーは、ノートの落下速度の倍率を指定できる
     - 最遅0.5倍 〜 最速3.0倍とし、適当な幅に刻んだ倍率値を指定できるものとする(ユーザーに直接倍率を入力させない)
   - [x] ゲームスタートボタン押下後すぐにゲームを始めるのではなく、準備時間を設けること
     - とりあえず3秒待機(ユーザーは待機時間を延ばせない)でよしとする
 - ゲーム中
   - [x] ゲーム中、画面下部に存在する判定ラインに対して、画面上部から真下方向に時間経過に応じてノートを近づける
     - 表示するのは判定ラインに近いノートのみでOK、1曲に設定されている全てのノートを1画面内に表示する必要はない
     - 判定ラインは水平方向に伸びる線分とする、判定ラインの形状等の変更対応は別ストーリーで管理する
   - [x] ゲーム中、プレイヤーが判定ライン付近のノートを左クリックすると、判定ラインとの近さに応じて結果を判定する
   - [x] 結果判定時、その結果をプレイヤーが確認しやすい位置に一定時間表示する
     - 確認しやすい位置とは、プレイヤーが目線をノートおよび判定ラインの周辺からそらさずに見られる位置とする
   - [x] 結果が判定されたノートは、結果判定直後に画面から消す(非表示にする)
   - [x] 結果を判定する際、判定ラインから遠すぎる場合は未判定扱いにする
   - [x] 判定ラインを通り過ぎて一定時間が経っても結果を判定されていないノートは、見逃したと判定する
   - [x] ゲーム中、常に現在の結果を表示する
   - [x] ゲーム中、プレイヤーがノートを拾うための入力として、左クリック以外にキーボード操作をサポートすること
   - [ ] ゲーム中、プレイヤーがノートを拾う意思を示す入力を行った場合、入力があったことをわかりやすく表示する
     - 入力位置に該当する一定範囲を光らせることとする、まずは判定ライン付近のみを光らせることで本ストーリー達成とする
     - この際ノートがちゃんと見えるように光らせること
   - [x] ゲーム中、プレイヤーがノートを拾った際、それとわかる効果音を再生すること
     - プレイヤーが見逃したノートに関して、効果音を再生しないこと
 - ゲーム終了
   - [x] ゲーム開始後、曲が終了するとゲームを終了し、合計結果を表示する
   - [x] ゲーム終了時、その旨を何かしらの手段でプレイヤーに伝える
     - 手段：画面表示とする
   - [x] ゲーム終了時、プレイ結果を保存する
 - ゲーム中断/再開機能
   - [x] ゲーム中、プレイヤーはゲームを中断することができる
   - [x] ゲーム中断後、プレイヤーはゲームを再開することができる
 - その他
   - [x] 管理担当者はwav形式の曲データを追加できなければならない
   - [ ] 管理担当者はmp3形式の曲データを追加できることが望ましい
   - [x] 管理担当者は曲毎にノートを外部ファイルで設定できなければならない
   - [ ] 管理担当者は、判定ラインの形状とノートが降ってくる方向を定めたモードに応じたノートデータを設定できる
   - [ ] 管理担当者は曲毎のノート設定を簡単に行えること
     - メモ：これはストーリー？要望？　いずれにせよこのままだと粒度が大きすぎるので分解が必要
