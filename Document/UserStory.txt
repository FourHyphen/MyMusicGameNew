 - ストーリーメモ
   - 残す目的：良いストーリー/悪いストーリーの勉強用
   - 「誰が」「何をすると」「結果どうなる」「何のためにこの機能が存在する」まで書くべき
     - 自分用なのでそこまでしない
 - 画面立ち上げ後
   - [x] 画面立ち上げ後、設定ファイルから読み込んだ曲リストが表示される
     - 少なくともjson形式をサポートすること
   - [x] 画面立ち上げ後、曲を選択するようプレイヤーに何かしらの手段で伝える
     - 手段：画面表示とする
   - [x] 曲リストから曲を選択しない状態でゲームスタートボタンを押すと、ゲームが始まらない
     - この際、何も表示(プレイヤーに通知)しないでOK
   - [ ] 曲リストから曲を選択する際、これまでのベスト結果を表示する
 - ゲーム開始時
   - [x] 曲リストから曲を選択した状態でゲームスタートボタンを押すと、ゲームが始まる
     - その際、何かしらの手段でプレイヤーにゲームが始まった旨を伝える
     - 手段：画面表示とする
   - [x] ゲーム開始時、プレイヤーが選択した曲を流し始める
 - ゲーム中
   - [ ] ゲーム中、ノートが表示され、時間が経つにつれて判定ラインに徐々に近づいていく
     - 表示するのは判定ラインに近いノートのみでOK、1曲に設定されている全てのノートを1画面内に表示する必要はない
   - [ ] ゲーム中、プレイヤーがノートを拾う意思を入力すると、判定ラインとの近さに応じて結果を判定する
   - [ ] 結果判定時、その結果をプレイヤーが確認しやすい位置に一定時間表示する
     - 確認しやすい位置とは、プレイヤーが目線をノートおよび判定ラインの周辺からそらさずに見られる位置とする
   - [ ] 結果が判定されたノートは、結果判定直後に画面から消す(非表示にする)
   - [ ] 結果を判定する際、判定ラインから遠すぎる場合は未判定扱いにする
   - [ ] 判定ラインを通り過ぎて一定時間が経っても結果を判定されていないノートは、見逃したと判定する
   - [ ] ゲーム中、常に現在の結果を表示する
 - ゲーム終了
   - [ ] ゲーム開始後、曲が終了するとゲームを終了し、合計結果を表示する
   - [ ] ゲーム終了時、その旨を何かしらの手段でプレイヤーに伝える
   - [ ] ゲーム終了時、プレイ結果を保存する
 - ゲーム中断/再開機能
   - [ ] ゲーム中、プレイヤーはゲームを中断することができる
   - [ ] ゲーム中断後、プレイヤーはゲームを再開することができる
 - その他
   - [x] 管理担当者はwav形式の曲データを追加できなければならない
   - [ ] 管理担当者はmp3形式の曲データを追加できることが望ましい
